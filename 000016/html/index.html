En esta oportunidad mostrare como usar el framework de desarrollo rapido:  Opexava. Es muy interesante este framework  ya que puede ser una solucion rapida para proyectos sin tiempo, para demos, incluso para proyectos empresariales. Como todo tiene sus ventajas y desventajas. Pero no esta demas conocerla.

Requisitos:

  1. Openxava Framework
  2. Librerias (ejb.jar - jta.jar)
  3. Servidor de base de datos Mysql
  4. Driver o Middleware Mysql
  5. Eclipse IDE

Como funciona?

Openxava crea una web completa a partir de entidades, clases, beans o pojos java :

Conceptos Basicos:

Componente de negocio
Las piezas fundamentales para crear una aplicación OpenXava son los componentes, en el contexto de OpenXava un componente de negocio es una clase Java
Controladores
Un controlador es un conjunto de acciones. Una acción es un botón o vínculo que el usuario puede pulsar.
Aplicación
Una aplicación OpenXava es un conjunto de módulos. Un módulo une un componente de negocio con uno o más controladores.

 

Pasos para desplegar una aplicacion openxava en el tomcat

  1. 1
    Creación del proyecto openxava.

 

Descargar de la web el zip de openxava, los descomprimimos y con el eclipse abrir la carpeta workspace:

En el Eclipse creamos un nuevo proyecto Java : Para este ejemplo usaremos "Empresa"
Vamos al proyecto OpenXavaPlantilla y ejecutamos CrearNuevoProyecto.xml

Nos pedirá un nombre para el proyecto : "Empresa"

En estos pasos, openxava ha convertido nuestro proyecto java en un proyecto web openxava listo para empezar.
Para visualizar estos cambios en el proyecto, seleccionamos el proyecto "Empresa" y pulsamos F5 para refrescar.

  1. 3
    Configuracion de la base de datos

 

Necesitamos cambiar solo estos archivos:

Persistence.xml

hibernate.cfg.xml

build.xml

Descargamos el driver de coneccion para nuestra base de datos. Para este ejemplo usaremos mysql.(Requisito numero 4)

Luego en el archivo build, en el tag : <target name="actualizarEsquema"> seteamos la ruta de este jar descargado:

Este paso es solo para crear las tablas en la base de datos. Cuando se despliegue la aplicación, este jar debe estar incluido en el war o en nuestro servidor de aplicaciones.

 

context.xml

Agregamos nuestro datasource en el archivo context.xml del servidor de aplicaciones(tomcat). Ejemplo : R:\servers\apache-tomcat-7.0.39\conf

Los archivos resultantes son:

  1. persistence.xml
  2. hibernate.cfg.xml
  3. build.xml
  4. context.xml

 

  1. 4
    Creación de las tablas en base de datos a partir del modelo

 

Hacemos lo siguiente:

1.- Creamos el esquema "empresa" en nuestra base de datos (mysql para este ejemplo). Este nombre debe ser el mismo que el usado en el persistence.xml, hibernate.xml y context.xml:

"jdbc:mysql://localhost:3306/empresa"

2.- Iniciamos nuestro mysql y ejecutamos la tarea "actualizar esquema" del build.xml:

Clic derecho en build.xml > run as > ant buil ... y seleccionamos "actualizar esquema"

Si todo resulta bien, en la consola se vera algo como:

https://gist.github.com/jrichardsz/6964217

 

  1. 5
    Generacion del artefacto o war

 

Openxava por defecto, tiene configurado un ant para poder deployar directamente el proyecto a el tomcat que viene en su zip:

Esto esta bien para una demo, pero en un entorno mas real, necesitaremos generar el war para poder desplegarlo o deployarlo en algun servidor de aplicaciones. Para ellos modificamos el archivo build.xml

 

Finalmente ejecutamos la tarea "desplegar war" de el build.xml asi: Clic derecho en build.xml > run as > ant buil ...

Si todo salio bien , en la consola se vera algo como esto :

https://gist.github.com/jrichardsz/6959755

 

Nota: Al generar el war de esta forma, 3 jars son necesarios:

ejb.jar, jta.jar y mysql-connector-java-5.1.18.jar (para este ejemplo es mysql)

La solucion es agregar estos jars al war ya generado : Abrir el war con el winrar , 7zip u otra herramienta y agregar los jars en : Empresa.war > WEB-INF\lib

o agregar estos jar a nivel del servidor: x:\servers\apache-tomcat-7.0.39\lib

  1. 4
    Desplegar el war en un servidor de aplicaciones (para este ejemplo tomcat)

 

Finalmente desplegamos el war como sabemos: copiamos el archivo Empresa.war a la carpeta

x:\servers\apache-tomcat-7.0.39\webapps

y ejecutamos el bat : startup.bat ubicado en :

x:\servers\apache-tomcat-7.0.39\bin

Si todo sale bien, no deberian haber errores en el log del tomcat.

 

  1. 5
    Lanzar la aplicación

 

Creamos la url de nuestra aplicación:

http://<host-servidor-aplicaciones>:<port-servidor-aplicaciones>/<nombre-proyecto-eclipse>/modules/<nombre-pojo-java>

Para nuestro ejemplo, quedaria asi :

http://localhost:8080/Empresa/modules/Cliente
